Train  Consist  Management  App   
The  Train  Consist  Management  App  is  a  console-based  Java  application  that  simulates  how  a  railway  
system
 
manages
 
a
 
train’s
 
consist,

The  application  supports:   
●  Passenger  bogies  (Sleeper,  AC  Chair,  First  Class)  with  seat  capacity  tracking  ●  Goods  bogies  (Rectangular,  Cylindrical)  with  cargo  type  and  safety  constraints  ●  Tracking  composition ,  capacity ,  cargo  types ,  and  safety  compliance  
Each  use  case  introduces  one  or  more  Java  concepts  through  a  realistic  railway  Scenario .  
UC6:  Map  Bogie  to  Capacity  (HashMap)  
 Drawback  of  UC5  Approach  In  UC5,  the  train  formation  stores  only  the  names  of  bogies  in  order  and  ensures  uniqueness.  
However,

Goal  
Associate  each  bogie  with  its  seating  or  load  capacity  using  a  key–value  mapping  structure.

Flow  
1.  User  creates  a  bogie-capacity  mapping.  2.  System  stores  bogie  names  as  keys  and  capacities  as  values.  3.  Entries  are  inserted  into  a  HashMap.  4.  System  iterates  through  the  map.  5.  Capacity  details  are  displayed  for  each  bogie.  6.  Program  continues.

Key  Requirements  
●  Create  a  HashMap<String,  Integer> to  store  bogie-capacity  information.  ●  Insert  capacity  values  for  bogies  such  as  Sleeper,  AC  Chair,  and  First  Class.  ●  Use  the  put() method  to  map  each  bogie  to  its  capacity.  ●  Iterate  over  the  map  using  entrySet().  ●  Display  each  bogie  along  with  its  corresponding  capacity.   Key  Benefits  
●  Models  real-world  attribute  mapping  in  software  systems.  ●  Introduces  key–value  data  representation  instead  of  flat  collections.  ●  Enables  fast  lookup  and  validation  of  bogie  properties.  ●  Prepares  students  for  enterprise  data  modeling  patterns.  ●  Builds  the  foundation  for  analytics  and  planning  use  cases  in  later  UCs.
